# S-019: Benchmark Comparative des Systèmes IA

## Résumé

**BUT:** Créer un **framework d'évaluation scientifique** pour déterminer quelle approche IA (RAG, MCP, long context, fine-tunage) est "meilleure" pour répondre aux questions d'antibioprophylaxie chirurgicale.

**LIVRABLE:** 
- Dataset de test standardisé (25 questions)
- Scripts d'évaluation automatique
- Rapport comparatif avec recommandations

---

## User Story

En tant que **chercheur/développeur**, je veux pouvoir:
1. Tester différents systèmes IA sur les mêmes questions
2. Évaluer précision + performance (latency, coûts, écologie)
3. Générer un rapport comparatif pour décider **quel stack intégrer dans la v2**

## Systèmes à tester

### 1. RAG Niveau 1 (PDF)

**Approche:** Retriever sur le PDF original RFE SFAR
- Ingester PDF → VectorDB (embed chaque section)
- Pour chaque question, retriever sections pertinentes
- Envoyer au LLM: context + question

**Métrique clé:** Latency (embedding + retrieval + LLM gen)

### 2. RAG Niveau 2 (Excel structuré)

**Approche:** Retriever sur l'export Excel (data/rfe.json)
- Ingester JSON → VectorDB (embed interventions + recommandations)
- Requête: "Trouvez interventions + prothèses hanche"
- Contexte structuré au LLM

**Métrique clé:** Coûts (moins de tokens = moins cher)

### 3. Long Context (Claude Opus)

**Approche:** Tout le texte RFE en context window
- Claude Opus 200k tokens
- Charger intégralement: PDF + JSON
- Zéro retrieval, juste génération

**Métrique clé:** Latency + Précision (moins d'oublis)

### 4. Serveur MCP

**Approche:** Model Context Protocol avec tools
- Tool 1: `search_interventions(name)` → JSON
- Tool 2: `get_recommendation(intervention_id)` → texte
- LLM appelle tools, synthétise

**Métrique clé:** Sourçage (réponses incluent citation)

### 5. LLM Fine-tuné

**Approche:** Fine-tune un modèle sur instruction set
- Dataset: Q-A pairs + recommandations RFE
- Fine-tune: Claude/GPT/Llama
- Tester sur questions test

**Métrique clé:** Précision (après tunage, moins hallucination)

## Métriques d'évaluation

Pour chaque système:

| Métrique | Calcul | Unité |
|----------|--------|-------|
| **Précision** | (réponses exactes) / 25 | % |
| **Latency** | temps moyen par question | s |
| **Tokens** | tokens utilisés (prompt + completion) | nb |
| **Coûts** | prix modèle * tokens | € |
| **Écologie** | émissions CO2 estimées | g CO2e |
| **Sourçage** | réponses avec citation | % |

### Scoring

```python
# Exact match (questions ouvertes)
def score_open(réponse_attendue, réponse_modèle):
    return réponse_attendue.lower().strip() == réponse_modèle.lower().strip()

# QCM
def score_qcm(lettre_attendue, réponse_modèle):
    return lettre_attendue.upper() in réponse_modèle.upper()
```
